# 🎓 Telegram Бот для Репетиторов

Бесплатный Telegram-бот для управления учениками, расписанием, домашними заданиями и оплатой занятий. Работает 24/7 на бесплатном хостинге PythonAnywhere.

## ✨ Возможности

### Для репетитора (администратора):
- ✅ **Управление учениками**: принятие заявок, присвоение никнеймов, удаление
- 📅 **Расписание**: постоянное расписание занятий, переносы отдельных уроков
- 📝 **Домашние задания**: добавление, удаление, автоматические уведомления ученикам
- 💰 **Оплата**: отслеживание оплаченных занятий, автоматический подсчет по дням занятий
- 📢 **Объявления**: рассылка сообщений всем ученикам
- ⏰ **Автоматические напоминания**: о занятиях и приближающейся оплате

### Для учеников:
- 📋 Просмотр своего расписания
- 📝 Список домашних заданий
- 💳 Информация об оплате
- 🔔 Уведомления о занятиях и новых заданиях

## 🚀 Быстрый старт

### Шаг 1: Получить токен бота

1. Открой Telegram и найди [@BotFather](https://t.me/BotFather)
2. Отправь команду `/newbot`
3. Придумай имя для бота (например: `My Tutor Bot`)
4. Придумай username (например: `mytutorbot` - должен заканчиваться на `bot`)
5. **Скопируй токен** - это длинная строка типа `1234567890:ABCdefGHIjklMNOpqrsTUVwxyz`
6. **Сохрани токен** - он понадобится!

### Шаг 2: Узнать свой Telegram ID

1. Найди в Telegram бота [@userinfobot](https://t.me/userinfobot)
2. Нажми **Start**
3. **Скопируй свой ID** (число типа `123456789`)
4. **Сохрани ID** - это твой идентификатор администратора!

### Шаг 3: Скачать файлы

1. Нажми зеленую кнопку **"Code"** на этой странице
2. Выбери **"Download ZIP"**
3. Распакуй архив
4. Найди файлы `complete_bot.py` и `main.py`

### Шаг 4: Настроить бота

Открой файл `complete_bot.py` любым текстовым редактором (Блокнот, Notepad++, VS Code) и найди эти строки в начале файла:

```python
BOT_TOKEN = "YOUR_BOT_TOKEN_HERE"  # Замени на свой токен!
ADMIN_ID = 123456789  # Замени на свой ID!
```

**Замени:**
- `YOUR_BOT_TOKEN_HERE` на токен который дал BotFather
- `123456789` на твой ID от @userinfobot

**Пример после замены:**
```python
BOT_TOKEN = "1234567890:ABCdefGHIjklMNOpqrsTUVwxyz"
ADMIN_ID = 987654321
```

**Сохрани файл!**

## 🌐 Запуск на PythonAnywhere (бесплатно, 24/7)

### 1️⃣ Регистрация

1. Зайди на [pythonanywhere.com](https://www.pythonanywhere.com)
2. Нажми **"Pricing & signup"** → **"Create a Beginner account"**
3. Заполни форму (это **бесплатно навсегда!**)
4. Подтверди email

### 2️⃣ Загрузка файлов

1. После входа нажми вкладку **"Files"** (вверху страницы)
2. Нажми кнопку **"Upload a file"**
3. Загрузи файлы:
   - `complete_bot.py`
   - `main.py`
4. Оба файла должны появиться в списке

### 3️⃣ Установка библиотек

1. Нажми вкладку **"Consoles"**
2. Нажми **"Bash"** (откроется черное окно - консоль)
3. **Скопируй и вставь** эту команду:

```bash
pip3.10 install --user python-telegram-bot==20.7 apscheduler pytz
```

4. Нажми **Enter** и подожди 1-2 минуты
5. В конце должно появиться: `Successfully installed ...`

### 4️⃣ Запуск бота

В той же консоли введи:

```bash
nohup python3.10 main.py > bot.log 2>&1 &
```

**Готово!** Бот теперь работает 24/7! 🎉

### 5️⃣ Проверка

Проверь что бот запущен:

```bash
ps aux | grep main.py
```

Если видишь строку с `python3.10 main.py` - всё работает! ✅

Теперь открой Telegram, найди своего бота и напиши `/start` - увидишь панель управления!

## 📱 Команды бота

### Для репетитора (администратора):

#### Управление учениками
```
/requests - Посмотреть запросы на регистрацию
/accept @username - Принять ученика
/reject @username - Отклонить запрос
/students - Список всех учеников
/remove Никнейм - Удалить ученика
/setnickname @username Никнейм - Задать никнейм
```

**Пример:**
```
/setnickname @ivan Макс
```

#### Расписание
```
/addschedule Никнейм день время
/removeschedule Никнейм день
/reschedule Никнейм старая_дата новая_дата новое_время
```

**Дни недели:** 0=Понедельник, 1=Вторник, 2=Среда, 3=Четверг, 4=Пятница, 5=Суббота, 6=Воскресенье

**Примеры:**
```
/addschedule Макс 1 14:00
(Добавить занятие по вторникам в 14:00)

/reschedule Макс 18.02.2026 19.02.2026 15:00
(Перенести занятие с 18 февраля на 19 февраля в 15:00)
```

#### Домашние задания
```
/addhw Никнейм дата текст - Добавить задание
/deletehw Никнейм дата - Удалить задание
```

**Пример:**
```
/addhw Макс 20.02.2026 Решить задачи 1-10 из учебника
```

#### Предметы и настройки
```
/setsubject Никнейм тема - Установить предмет
/setrate Никнейм ставка - Установить ставку (в тенге)
/setduration Никнейм минуты - Установить длительность занятия
```

**Примеры:**
```
/setsubject Макс Математика SAT
/setrate Макс 5000
/setduration Макс 60
```

#### Оплата
```
/addpayment Никнейм тип количество
/clearpayment Никнейм - Очистить информацию об оплате
```

**Типы оплаты:** день, неделя, месяц

**Пример:**
```
/addpayment Макс месяц 8
(Ученик оплатил 8 занятий)
```

Бот автоматически рассчитает:
- Дату последнего оплаченного занятия (включительно)
- Дату следующей оплаты
- Дни будут совпадать с днями занятий!

#### Объявления
```
/announce текст - Отправить объявление всем ученикам
```

### Для учеников:

```
/start - Начать работу
/register - Отправить запрос на регистрацию
/schedule - Моё постоянное расписание
/thisweek - Расписание на этой неделе (с переносами)
/homework - Мои домашние задания
/info - Моя информация (предмет, ставка, длительность)
/payment - Информация об оплате
```

## ⏰ Автоматические уведомления

Бот отправляет уведомления автоматически:

### Напоминания о занятиях
- **Время:** Каждый день в 3:00 утра (время Алматы)
- **Кому:** Всем ученикам у кого занятие сегодня
- **Содержание:** Время занятия, предмет, информация о переносе (если есть)

### Напоминания об оплате
- **Время:** Каждый день в 10:00 утра (время Алматы)
- **Кому:** Ученикам у кого скоро заканчивается оплата
- **Когда отправляется:**
  - За 5 дней до следующей оплаты (если оплата "месяц")
  - За 1 день до следующей оплаты (если оплата "неделя")

### Обновление счетчика занятий
- **Время:** Каждый день в 23:00 (время Алматы)
- **Что делает:** Автоматически уменьшает счетчик оставшихся занятий после каждого урока

## 🌍 Смена часового пояса

По умолчанию используется часовой пояс **Asia/Almaty**.

Чтобы изменить часовой пояс, найди в файле `complete_bot.py` эту строку:

```python
ALMATY_TZ = pytz.timezone('Asia/Almaty')
```

Замени на свой часовой пояс:

```python
# Москва
ALMATY_TZ = pytz.timezone('Europe/Moscow')

# Киев
ALMATY_TZ = pytz.timezone('Europe/Kiev')

# Нью-Йорк
ALMATY_TZ = pytz.timezone('America/New_York')

# Лондон
ALMATY_TZ = pytz.timezone('Europe/London')
```

[Полный список часовых поясов](https://en.wikipedia.org/wiki/List_of_tz_database_time_zones)

## 🐛 Решение проблем

### Бот не отвечает в Telegram

**Проверь что бот запущен:**
```bash
ps aux | grep main.py
```

Если ничего не показывает - запусти бота снова:
```bash
nohup python3.10 main.py > bot.log 2>&1 &
```

**Посмотри логи на ошибки:**
```bash
tail -50 bot.log
```

### Ошибка "Module not found"

**Установи библиотеки заново:**
```bash
pip3.10 install --user python-telegram-bot==20.7 apscheduler pytz
```

### Бот падает сразу после запуска

**Возможные причины:**

1. **Неправильный токен** - проверь `BOT_TOKEN` в файле `complete_bot.py`
2. **Неправильный ADMIN_ID** - проверь `ADMIN_ID` в файле `complete_bot.py`
3. **Синтаксическая ошибка** - посмотри логи: `cat bot.log`

### База данных не создается

**Проверь права доступа:**
```bash
ls -la tutor_bot.db
```

Если файла нет - он создастся при первом запуске. Подожди 10 секунд после запуска.

### Не приходят уведомления

**Проверь что планировщик работает:**

Посмотри логи при запуске бота:
```bash
cat bot.log | grep "Планировщик"
```

Должно быть: `✅ Планировщик запущен`

## 🔧 Технические детали

### Требования

- Python 3.10+
- Библиотеки:
  - `python-telegram-bot==20.7`
  - `apscheduler`
  - `pytz`

### Структура проекта

```
tutor-bot/
│
├── complete_bot.py    # Основной код бота (все функции)
├── main.py            # Файл запуска (регистрация команд)
├── tutor_bot.db       # База данных SQLite (создается автоматически)
└── README.md          # Эта инструкция
```

### База данных (SQLite)

**Таблицы:**

1. `registration_requests` - запросы на регистрацию
2. `students` - ученики
3. `permanent_schedule` - постоянное расписание
4. `schedule_changes` - переносы занятий
5. `homework` - домашние задания
6. `payments` - история платежей
7. `sent_notifications` - отправленные уведомления
