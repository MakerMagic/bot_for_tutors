# 📚 Примеры использования

Реальные сценарии работы с ботом.

## 🎯 Сценарий 1: Новый ученик

### Шаг 1: Ученик отправляет запрос

**Ученик (Иван):**
```
/register
```

**Бот ученику:**
```
✅ Твой запрос отправлен преподавателю!

Ожидай подтверждения. Тебе придёт уведомление.
```

### Шаг 2: Тебе приходит уведомление

**Бот тебе:**
```
🔔 Новый запрос на регистрацию!

👤 Имя: Иван Петров
ID: @ivan_petrov

/accept @ivan_petrov - Принять
/reject @ivan_petrov - Отклонить
```

### Шаг 3: Ты принимаешь ученика

**Ты:**
```
/accept @ivan_petrov
```

**Бот тебе:**
```
✅ Ученик @ivan_petrov добавлен!

Не забудь задать никнейм:
/setnickname @ivan_petrov Имя
```

### Шаг 4: Задаешь никнейм

**Ты:**
```
/setnickname @ivan_petrov Иван
```

**Бот тебе:**
```
✅ Никнейм для @ivan_petrov: Иван

Теперь можешь обращаться как:
/addhw Иван ...
```

### Шаг 5: Настраиваешь ученика

**Ты:**
```
/setsubject Иван Математика ЕГЭ
/setrate Иван 3000
/setduration Иван 60
```

**Бот:**
```
✅ Тема для Иван: Математика ЕГЭ
✅ Ставка для Иван: 3000 тенге
✅ Длительность для Иван: 60 минут
```

### Шаг 6: Добавляешь расписание

**Ты:**
```
/addschedule Иван 1 16:00
/addschedule Иван 4 16:00
```

**Бот:**
```
✅ Добавлено: Иван - Вторник в 16:00
✅ Добавлено: Иван - Пятница в 16:00
```

## 💰 Сценарий 2: Ученик оплатил месяц

### Ситуация

Ученик Иван занимается 2 раза в неделю (вторник, пятница).
Он оплатил месяц - 8 занятий.
Сегодня 10 февраля 2026 (понедельник).

### Ты добавляешь оплату

**Ты:**
```
/addpayment Иван месяц 8
```

### Бот считает автоматически

**Бот тебе:**
```
✅ Оплата добавлена!

👤 Иван
💳 Тип: месяц
📊 Занятий: 8
✅ Оплачено до (включительно): 07.03.2026 (Пятница)
⏰ Следующая оплата с: 10.03.2026 (Вторник)
```

**Как бот посчитал:**
- Начинаем с 10 февраля (понедельник)
- Ближайшее занятие: 11 февраля (вторник) - 1 занятие
- Потом 13 февраля (пятница) - 2 занятия
- Потом 18 февраля (вторник) - 3 занятия
- Потом 20 февраля (пятница) - 4 занятия
- Потом 25 февраля (вторник) - 5 занятий
- Потом 27 февраля (пятница) - 6 занятий
- Потом 04 марта (вторник) - 7 занятий
- Потом 07 марта (пятница) - **8 занятий** ← последнее оплаченное
- Следующее занятие 10 марта (вторник) - уже новая оплата

**Ученик видит:**
```
💰 Информация о зарплате:

📋 Тип оплаты: месяц
📊 Осталось занятий: 8
✅ Оплачено до (включительно): 07.03.2026 (Пятница)
⏰ Следующая оплата: 10.03.2026 (Вторник)
```

## 📅 Сценарий 4: Перенос занятия

### Ситуация

У Ивана занятие в пятницу 20 февраля в 16:00.
Нужно перенести на субботу 21 февраля в 14:00.

**Ты:**
```
/reschedule Иван 20.02.2026 21.02.2026 14:00
```

**Бот тебе:**
```
✅ Перенос создан!

Иван
С 20.02.2026 (Пятница)
На 21.02.2026 (Суббота) в 14:00
```

**Иван автоматически получает:**
```
📅 ПЕРЕНОС ЗАНЯТИЯ

Занятие 20.02.2026 (Пятница)
перенесено на 21.02.2026 (Суббота) в 14:00
```

### Иван проверяет расписание на неделю

**Иван:**
```
/thisweek
```

**Бот Ивану:**
```
📅 Расписание на этой неделе:

▪️ 18.02.2026 (Вторник) в 16:00
▪️ 20.02.2026 (Пятница) в 16:00 → ПЕРЕНОС на 21.02.2026 (Суббота) в 14:00

⚠️ На этой неделе есть переносы!
```

## 💡 Полезные советы

### 1. Используй никнеймы вместо @username

**Плохо:**
```
/addhw @ivan_petrov_super_long_username 20.02.2026 Задание
```

**Хорошо:**
```
/setnickname @ivan_petrov_super_long_username Ваня
/addhw Ваня 20.02.2026 Задание
```

### 2. Одно собщение для всех настроек

Вместо отдельных команд:
```
/setsubject Иван Математика
/setrate Иван 3000
/setduration Иван 60
```

Можно отправить всё сразу:
```
/setsubject Иван Математика ЕГЭ
/setrate Иван 3000
/setduration Иван 60
```

### 3. Проверяй расписание ученика перед добавлением оплаты

```
/viewtasks Иван
```

Убедись что у ученика есть постоянное расписание перед `/addpayment`.

### 4. Делай резервные копии базы раз в месяц

На PythonAnywhere: Files → `tutor_bot.db` → Download
